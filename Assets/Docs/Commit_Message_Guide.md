# Project R Commit Message Guide

Project R에서 커밋하고 푸시할 때는 이 문서의 형식을 따른다.

## 기본 형식

```text
작업 묶음 제목

- 큰 기능 줄기 요약
- 큰 기능 줄기 요약
- 큰 기능 줄기 요약
```

## 제목 작성 규칙

- 제목은 이번 커밋의 전체 작업 묶음을 한 줄로 요약한다.
- 제목은 한국어를 기본으로 작성한다.
- 제목 끝에는 마침표를 붙이지 않는다.
- 너무 넓은 표현보다 실제 작업 범위가 드러나는 표현을 쓴다.

## 본문 작성 규칙

- 본문은 기능 하나의 전체적인 줄기를 한 줄로 요약한다.
- 각 줄은 `- `로 시작하는 bullet 형식을 사용한다.
- 세부 구현 방식, 함수명, 변수명, 수치, 파일명은 꼭 필요한 경우가 아니면 쓰지 않는다.
- 같은 목적의 세부 변경들은 하나의 큰 줄기로 묶는다.
- 구현, 수정, 적용, 정리, 문서화처럼 작업 성격이 드러나는 동사로 끝낸다.
- 커밋에 씬, 문서, 설정 변경이 포함되면 큰 줄기 단위로 함께 기록한다.

## 좋은 예시

```text
플레이어·카메라·아이템 프로토타입 구현

- 플레이어 스탯 구현
- 플레이어 이동 및 대시 구현
- 플레이어 추적 카메라 구현
- 프로토타입 타일맵 및 이동 가능 영역 구현
- 자동 루팅 아이템 및 이동속도 버프 아이템 구현
- 아이템 루팅용 레이어 구분 적용
- 캐릭터, 아이템, 타일맵 프로토타입 씬 구성
- 최종 개발 씬에 프로토타입 기능 통합
```

## 피해야 할 예시

```text
플레이어 카메라 프로토타입 기능 기록

- PlayerStats의 temporaryMoveSpeedMultiplier를 1.2로 변경하고 3초 타이머를 사용한다.
- SmoothCameraFollow2D의 followSharpness, cameraMoveSpeed, followLerpCurve, snapDistance 값을 조절한다.
```

위 예시는 너무 구체적이거나 커밋의 작업 줄기가 잘 드러나지 않는다.

## 이번 프로젝트의 기준

- 커밋 메시지는 작업자가 나중에 히스토리를 훑을 때 큰 기능 흐름을 빠르게 파악하는 용도다.
- 자세한 구현 설명은 코드, 주석, 문서, PR 설명에 남기고 커밋 메시지는 큰 줄기만 남긴다.
- Codex가 Project R에서 커밋하거나 푸시할 때는 먼저 이 문서를 기준으로 메시지를 작성한다.
